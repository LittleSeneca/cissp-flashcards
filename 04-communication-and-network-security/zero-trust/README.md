# Zero Trust Architecture (ZTA)

This section covers the principles and components of Zero Trust Architecture as defined in NIST SP 800-207.

## Key Concepts
- **"Never Trust, Always Verify"**: The core tenet of Zero Trust.
- **Identity-Centric**: Access is based on user and device identity, not network location.
- **Policy Decision Point (PDP)**: The component that decides whether to grant access.
- **Policy Enforcement Point (PEP)**: The component that executes the decision made by the PDP.
- **Continuous Verification**: Ongoing assessment of trust throughout the session.
- **Least Privilege**: Granting only the minimum access necessary.

## NIST SP 800-207
The authoritative guide on Zero Trust, defining the architecture, components, and deployment models.
